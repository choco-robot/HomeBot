# HomeBot Plus 升降平台集成说明

## 概述

本文档描述 HomeBot Plus 版本中升降平台的集成方式。升降平台由两个 ST3215 舵机（ID 10 和 11）驱动，通过丝杆传动实现垂直升降。

## 坐标系定义

**新坐标系** (推荐):
- **最高点** = `0 mm`
- **向下** = 负方向
- **最低点** = `-stroke_length mm` (如 `-200mm`)

**坐标转换**:
```python
# 高度到步数 (最高点为参考零点)
steps = -offset_from_top * 4096 / lead / ratio / angle_resolution

# 步数到高度  
height = -offset_from_top
```

## 机械参数

当前配置：
- **丝杆导程 (lead)**: 20mm/转
- **传动比 (gear_ratio)**: 1:1
- **角度分辨率**: 4
- **行程 (stroke_length)**: 200mm
- **步数计算**: `steps = -offset(mm) * 4096 / 20 / 1 / 4 = -offset * 51.2`

例如：
- 0mm (最高点) = 0 步
- -100mm (中点) = -5120 步
- -200mm (最低点) = -10240 步

## 零点初始化（自动找零）

### 原理
上电后通过检测舵机电流/负载判断机械限位，自动设定坐标零点。

### 流程
1. 向配置方向（向上或向下）缓慢运动
2. 实时监测舵机电流
3. 电流超过阈值时停止，判定为碰到限位
4. 稍微回退释放机械压力
5. 将该位置设为坐标零点

### 配置
```python
@dataclass
class LiftPlatformConfig:
    # ... 机械参数 ...
    
    # 零点初始化参数
    auto_homing_on_startup: bool = True   # 启动时自动找零
    homing_direction: str = "up"          # 找零方向: "up"或"down"
    homing_speed: int = 300               # 找零速度 (不宜过快)
    homing_current_threshold: int = 300   # 电流阈值 (mA)
    homing_timeout: float = 15.0          # 超时时间(秒)
    homing_backoff_steps: int = 100       # 回退步数 (释放压力)
```

### 校准电流阈值

首次使用需要校准电流阈值：

```bash
cd software/src
python -m tests.test_lift_homing --calibrate
```

根据输出结果调整 `homing_current_threshold` 配置。

## API 使用

### 1. 机械臂服务指令

通过 ZeroMQ 向机械臂服务发送指令：

```python
import zmq

context = zmq.Context()
socket = context.socket(zmq.REQ)
socket.connect("tcp://localhost:5557")

# 控制升降平台到 -100mm (中点)
socket.send_json({
    "source": "web",
    "priority": 1,
    "lift": -100.0,       # 新坐标系: 负值表示向下
    "speed": 0
})
response = socket.recv_json()
```

### 2. 同时控制机械臂和升降平台

```python
socket.send_json({
    "source": "web",
    "priority": 1,
    "joints": {           # 机械臂关节角度
        "base": 0,
        "shoulder": 45,
        "elbow": 90
    },
    "lift": -50.0,        # 升降平台高度 (负值=向下)
    "speed": 1000
})
```

### 3. 查询状态

```python
socket.send_json({
    "source": "web",
    "priority": 1,
    "query": True
})
response = socket.recv_json()
# response 包含:
# - joint_states: 关节状态字典
# - lift_height: 当前升降高度 (新坐标系，0为最高点)
```

### 4. 使用客户端类

```python
from services.motion_service.chassis_arbiter import ArmArbiterClient

client = ArmArbiterClient("tcp://localhost:5557")

# 控制升降平台到最低点
response = client.send_lift_command(
    height=-200.0,         # 新坐标系: -200 = 最低点
    source="web",
    priority=1
)
print(f"当前高度: {response.lift_height}mm")
```

## 注意事项

1. **首次使用必须校准电流阈值**
   - 运行 `python -m tests.test_lift_homing --calibrate`
   - 根据结果调整 `homing_current_threshold`

2. **找零方向选择**
   - `up`: 向上找最高点（推荐，通常机械结构更稳定）
   - `down`: 向下找最低点

3. **坐标系理解**
   - 0mm = 最高点
   - -100mm = 向下100mm
   - -200mm = 最低点

4. **急停处理**
   - 触发急停后升降平台会立即停止
   - 复位时升降平台会回到最高点 (0mm)

## 故障排查

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 找零超时 | 电流阈值设置过高 | 降低 `homing_current_threshold` |
| 误检测限位 | 电流阈值设置过低 | 提高 `homing_current_threshold` |
| 高度不准确 | 机械参数错误 | 检查 `lead`, `gear_ratio`, `stroke_length` |
| 坐标方向反 | 步数符号错误 | 检查 `_height_to_steps` 转换 |
