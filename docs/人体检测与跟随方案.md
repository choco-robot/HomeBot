## 1. 功能概述

实现机器人的**人体检测**和**自动跟随**功能：
- **人体检测**：实时识别画面中的人体位置
- **目标跟踪**：稳定跟踪特定目标（避免频繁切换）
- **自动跟随**：根据目标位置控制底盘移动，保持固定距离

## 3. 技术方案选型

### 3.1 人体检测方案对比

| 方案 | 速度 | 精度 | CPU占用 | 模型大小 | 推荐度 |
|------|------|------|---------|----------|--------|
| **YOLO26-nano** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 极低 | 2.4MB | ⭐⭐⭐⭐⭐ |
| YOLO11-nano | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 低 | 2.6MB | ⭐⭐⭐⭐ |
| YOLOv8-nano | ⭐⭐⭐ | ⭐⭐⭐ | 低 | 3.2MB | ⭐⭐⭐ |
| MediaPipe | ⭐⭐⭐⭐ | ⭐⭐ | 极低 | - | ⭐⭐ |

**最终选择**: **YOLO26-nano** (2026年1月发布)
- **模型大小**: ~2.4MB（比YOLO11小8%，比YOLOv8小25%）
- **CPU速度**: 38.9ms/帧，比YOLO11快**43%**
- **精度**: 40.9% mAP（COCO），比YOLO11高1.4%
- **边缘优化**: 移除DFL模块，无NMS后处理，部署更简单

### 3.2 YOLO26 关键优势

| 特性 | YOLO26 | 说明 |
|------|--------|------|
| **端到端推理** | ✅ NMS-free | 直接输出结果，无需后处理，延迟更低 |
| **DFL移除** | ✅ | 架构简化，支持更多导出格式 |
| **小目标检测** | ✅ ProgLoss+STAL | 专门优化小目标检测精度 |
| **量化稳定** | ✅ | FP16/FP32精度一致，适合INT8部署 |
| **MuSGD优化器** | ✅ | 训练更稳定，泛化能力更强 |

### 3.3 目标跟踪方案

| 方案 | 特点 | 适用场景 |
|------|------|----------|
| **IoU匹配** | 简单、无状态 | 目标不重叠时 |
| **卡尔曼滤波** | 预测运动轨迹 | 目标短暂遮挡 |
| **DeepSORT** | 外观+运动特征 | 多目标、复杂场景 |

**推荐方案**: IoU匹配 + 卡尔曼滤波（混合）
- 轻量实现，适合树莓派/Jetson
- 处理简单跟随场景足够

### 3.4 跟随控制算法

**控制策略**：视觉伺服（Visual Servoing）

```python
# 控制逻辑伪代码
def compute_velocity(target_bbox, frame_width, frame_height):
    """
    根据目标位置计算底盘速度
    目标：将目标保持在画面中央，保持固定距离
    """
    cx, cy = get_bbox_center(target_bbox)
    area = get_bbox_area(target_bbox)
    
    # 1. 水平控制（角速度 vz）
    # 目标在画面左侧 → 左转，右侧 → 右转
    error_x = cx - frame_width / 2
    vz = Kp_angular * error_x  # P控制器
    
    # 2. 距离控制（线速度 vx）
    # 目标太小 → 靠近，太大 → 后退
    target_area = (frame_width * 0.3) * (frame_height * 0.5)  # 期望目标大小
    error_area = target_area - area
    vx = Kp_linear * error_area
    
    # 3. 速度限制
    vx = clamp(vx, -MAX_LINEAR_SPEED, MAX_LINEAR_SPEED)
    vz = clamp(vz, -MAX_ANGULAR_SPEED, MAX_ANGULAR_SPEED)
    
    return vx, 0, vz  # vy = 0（不前移不后退）
```

#### 4.2.4 HumanFollowApp (主应用)

```python
class HumanFollowApp:
    """人体跟随主应用"""
    
    def __init__(self):
        # 初始化视觉订阅
        self.vision_sub = VisionSubscriber()
        # 初始化底盘客户端
        self.chassis_client = ChassisClient()
        # 初始化检测器、跟踪器、控制器
        self.detector = HumanDetector()
        self.tracker = TargetTracker()
        self.controller = FollowController()
        
    def run(self):
        """主循环"""
        while running:
            # 1. 获取图像
            frame_id, frame = self.vision_sub.read_frame()
            
            # 2. 检测人体
            detections = self.detector.detect(frame)
            
            # 3. 更新跟踪
            target = self.tracker.update(detections)
            
            # 4. 计算速度
            if target:
                vx, vy, vz = self.controller.compute_velocity(target, frame.shape)
                # 5. 发送控制命令
                self.chassis_client.send_velocity(vx, vy, vz, source="auto")
            else:
                # 目标丢失，停止或搜索
                self.chassis_client.stop()
```

## 6. 异常处理与安全机制

### 6.1 异常场景处理

| 场景 | 处理策略 |
|------|----------|
| **目标丢失** | 停止移动，原地旋转搜索或等待 |
| **多目标** | 按策略选择（中央/最大/最近） |
| **目标遮挡** | 卡尔曼滤波预测，最多持续N帧 |
| **通信中断** | 停止底盘，记录日志 |
| **检测失败** | 重试，连续失败则停止 |

### 6.2 安全机制

1. **速度限制**：配置最大线速度和角速度上限
2. **急停响应**：监听紧急停止信号，立即释放控制权
3. **超时保护**：通信超时自动停止底盘
4. **边界检测**：结合视觉避障，避免碰撞

## 7. 性能优化

### 7.1 YOLO26 推理优化

| 优化手段 | 说明 | 预期提升 |
|----------|------|----------|
| **输入分辨率降低** | 320x320 替代 640x640 | 4x速度提升 |
| **FP16半精度** | TensorRT/ONNX FP16 | 2x速度提升 |
| **NCNN** (ARM) | 树莓派/手机专用格式 | 2-3x速度提升 |
| **批量推理** | 一次处理多帧 | 提高吞吐量 |

### 7.2 各平台预期性能

| 平台 | 格式 | 分辨率 | 预期FPS | 延迟 |
|------|------|--------|---------|------|
| 树莓派5 | NCNN | 320x320 | 25-30 | ~35ms |
| Jetson Nano | TensorRT FP16 | 320x320 | 20-25 | ~45ms |
| Jetson Orin Nano | TensorRT FP16 | 640x640 | 40-50 | ~20ms |
| x86 CPU | ONNX | 320x320 | 15-20 | ~60ms |

### 7.3 控制优化

- **速度平滑**：使用低通滤波或滑动平均，避免速度突变
- **PID控制**：将P控制器升级为PID，减少抖动

## 12. 附录：YOLO 版本对比

### 12.1 YOLO26 vs YOLO11 vs YOLOv8

| 指标 | YOLO26-nano | YOLO11-nano | YOLOv8-nano | 胜出 |
|------|-------------|-------------|-------------|------|
| **mAP** | 40.9% | 39.5% | 37.3% | YOLO26 |
| **CPU速度** | 38.9ms | 56.1ms | 80.4ms | YOLO26 |
| **参数量** | 2.4M | 2.6M | 3.2M | YOLO26 |
| **FLOPs** | 5.4B | 6.5B | 8.7B | YOLO26 |
| **NMS-free** | ✅ | ❌ | ❌ | YOLO26 |
| **发布时间** | 2026-01 | 2024-10 | 2023-01 | - |

### 12.2 选择YOLO26的理由

1. **最快CPU推理**: 比YOLO11快43%，比YOLOv8快107%
2. **更高精度**: 在更小模型下实现更高mAP
3. **无NMS**: 端到端推理，降低延迟和部署复杂度
4. **边缘优化**: 专为嵌入式和移动设备设计
5. **API兼容**: 与YOLOv8/YOLO11完全相同的Python接口
